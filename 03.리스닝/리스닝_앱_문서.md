## 1. 기획 배경 및 목표

### 배경
- 중1 영어 학생이 영어듣기능력평가 기출문제를 자율 학습할 수 있는 환경 필요
- 기존 유튜브 영상(풀영상) 시청 방식은 문제만 풀기 불편하고 오답 확인·스크립트 학습이 어려움
- 인터넷 없이도 사용 가능한 **오프라인 단일 HTML 파일** 형태로 구축

### 목표
1. **듣기 평가 환경 재현** — 실제 시험처럼 음성 재생 + 문제 풀기
2. **자기주도 학습** — 풀이 후 정답·오답·스크립트 확인
3. **진도 관리** — 회차별 진행 상태 로컬 저장
4. **확장성** — 회차 추가 시 JSON 데이터 + 음성 파일만 준비하면 됨

## 5. 앱 화면 구성 및 플로우

```
홈 화면
  └─▶ [시작] → 평가 화면 (exam mode)
                └─▶ [완료] → 결과 화면 (result mode)
                               ├─▶ [다시 풀기] → 평가 화면 (초기화)
                               └─▶ [홈으로] → 홈 화면

홈 화면 (완료 회차)
  └─▶ [결과 보기] → 결과 화면 (바로 진입)
```

### 5-1. 홈 화면
- 회차 카드 목록 (진행률 프로그레스 바 + 정답 수)
- 미시작: `시작` 버튼
- 진행 중: `계속하기` 버튼 + X/20 정답 표시
- 완료: `결과 보기` 버튼 (초록색)

### 5-2. 평가 화면 (시험 중)
- **상단 고정**: 음성 플레이어 (재생/일시정지, ±10초 이동, 0.8×/1×/1.25×/1.5× 배속)
- **진행 바**: `X/20 답변` (정답 수 미표시 — 시험 중 피드백 없음)
- **번호 격자**: 1~20번 버튼, 답변 완료 문항은 파란색
- **문제 카드**: 답안 선택 시 파란색 하이라이트만 표시, 정답/오답 색상 없음
- **하단**: `✓✓ 완료 — 결과 보기` 버튼 (전체 너비)

> 미응답 문항이 남아있을 때 완료를 누르면 확인 다이얼로그 표시

### 5-3. 결과 화면 (채점 후)
- **점수 카드**: X/20 정답, % 표시, 프로그레스 바
- **문제별 결과**: 정답(초록)/오답(빨강) 표시, 선택 답안·정답 강조
- **스크립트**: 모든 문항에 대화 스크립트 기본 표시
- **하단 버튼**: `다시 풀기` / `홈으로`

## 8. PDF → 데이터 파싱 파이프라인

### 8-1. 문제지 구조 특징
- A4 **2단 레이아웃**: Q1~Q11 (왼쪽), Q12~Q20 (오른쪽)
- Q1~Q3: 그림 보기 (임베디드 이미지 5개씩)
- Q14: 지도/표 형식 (일부 회차)
- Q19~Q20: 묶음 문항 (`【19-20】` 헤더)

### 8-2. 텍스트 추출 (pdfplumber)
```python
import pdfplumber

with pdfplumber.open("문제지.pdf") as pdf:
    for page in pdf.pages:
        w, h = page.width, page.height
        left  = page.crop((0, 0, w/2, h)).extract_text()   # Q1~11
        right = page.crop((w/2, 0, w, h)).extract_text()   # Q12~20
```
- 2단 분리로 컬럼 간 텍스트 혼용 문제 해결

### 8-3. 이미지 추출 (pdfimages + Pillow)
```bash
pdfimages -j 문제지.pdf /tmp/imgs/q
```
```python
from PIL import Image

# 면적 기반 필터링 (아이콘/배경 제외)
areas = [img.width * img.height for img in images]
median = sorted(areas)[len(areas)//2]
valid = [img for img in images if median*0.3 <= img.width*img.height <= median*12]

# Q1~Q3 각 5개씩 그룹핑
q1_imgs = valid[0:5]
q2_imgs = valid[5:10]
q3_imgs = valid[10:15]
```

### 8-4. 파싱 시 발생한 주요 문제와 해결

| 문제 | 원인 | 해결 |
|------|------|------|
| Q12~Q20 선택지에 Q1~Q11 텍스트 혼용 | 2단 레이아웃을 단일 추출 시 컬럼 섞임 | `page.crop()`으로 좌/우 분리 추출 |
| 이미지 필터링 오류 (2019: 너무 큰 면적 기준) | 연도별 이미지 해상도 편차 큼 (117px~945px) | 중앙값 기준 상대 필터링 (×0.3 ~ ×12) |
| 작은 로고 아이콘이 Q1 보기에 섞임 (2023) | 로고 이미지 크기가 유효 이미지와 유사 | 중앙값 하한 필터로 제거 |
| Q11 5번 선택지에 시험지 헤더 텍스트 붙음 | 오른쪽 컬럼 분리가 Q11 마지막 줄에서 불완전 | 정규식으로 `공동 주관 영어듣기능력평가...` 패턴 제거 |
| Q18 5번 선택지에 Q19-20 묶음 헤더 붙음 | `【19-20】대화를 듣고...` 가 Q18 마지막 줄에 이어짐 | 정규식으로 `【\d+-\d+】...` 패턴 제거 |
| Q20 5번 선택지에 저작권 고지 붙음 | 페이지 하단 저작권 텍스트 연결 | `ⓒ본 평가 문항의 저작권...` 패턴 제거 |
| Q19/20 문항이 앱에 렌더링 안 됨 | `buildDialogueContext(q)`가 `q.dialogueLines` 없을 때 JS 크래시 | `if (q.dialogueLines)` 조건부 처리 + `q.script` 기반 대화 렌더링 추가 |

## 11. 주요 JS 함수 목록

| 함수 | 역할 |
|------|------|
| `openSession(sid)` | 회차 진입 (완료된 회차는 결과 화면으로) |
| `renderSession(session)` | 평가 화면 렌더링 (격자 + 문제 카드) |
| `buildQCard(q, selected, sid)` | 개별 문제 카드 생성 (평가 모드) |
| `selectAnswer(sid, qNum, idx)` | 답안 선택 저장 + 카드 갱신 |
| `updateScore(session)` | 진행 바 갱신 (답변 수만 표시) |
| `submitSession()` | 완료 처리 → 결과 화면 전환 |
| `renderResult(session)` | 결과 화면 렌더링 |
| `buildResultQCard(q, selected)` | 개별 결과 카드 (정답/오답 + 스크립트) |
| `retrySession()` | 초기화 후 재시험 |
| `initAudioPlayer(session)` | 음성 플레이어 초기화 |
| `buildDialogueContext(q)` | 대화 완성형 문제 컨텍스트 렌더링 |
